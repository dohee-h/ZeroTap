# ZeroTap 게임 개발 블루프린트

## 1. 프로젝트 개요

**ZeroTap**은 사용자의 반응 속도를 테스트하는 간단하고 중독성 있는 웹 게임입니다. 임의의 순간에 화면에 나타나는 시각적 신호(색상 변화)를 사용자가 얼마나 빨리 터치(클릭)하는지를 측정합니다. 최고 기록을 경신하면 자신의 이름을 남길 수 있습니다. 이 프로젝트는 모바일 우선 접근 방식으로 반응형으로 디자인되며, 다크 모드와 라이트 모드를 모두 지원하여 사용자 경험을 향상시킵니다.

**핵심 문구:**
- "No Time to Think"
- "How Fast Are You?"

## 2. 기능 명세

### 2.1. 핵심 게임 플레이
- **게임 시작:** 사용자가 '시작' 영역을 클릭하면 게임이 준비 상태에 들어갑니다.
- **신호 발생:** "준비하세요..."와 같은 메시지가 표시된 후, 1초에서 5초 사이의 무작위 시간이 지나면 화면 전체 배경색이 갑자기 바뀝니다.
- **반응 속도 측정:** 배경색이 바뀐 순간부터 사용자가 화면을 클릭하는 순간까지의 시간을 밀리초(ms) 단위로 측정합니다.
- **조기 클릭 방지:** 신호가 발생하기 전에 사용자가 클릭하면 "너무 빨라요!" 메시지를 표시하고 게임을 다시 시작합니다.
- **결과 표시:** 측정된 반응 속도를 화면에 표시합니다.

### 2.2. 최고 기록 시스템
- **기록 저장:** 최고 기록은 브라우저의 `localStorage`에 저장되어 세션을 유지해도 사라지지 않습니다.
- **기록 경신:** 현재 기록이 저장된 최고 기록보다 빠를 경우, 사용자에게 이름을 입력하라는 메시지를 표시합니다.
- **기록 표시:** 이름이 제출되면 "최고 기록 - [이름]: [시간]ms" 형식으로 화면 하단에 표시됩니다.

### 2.3. UI/UX
- **테마:**
    - 다크 모드와 라이트 모드를 전환할 수 있는 토글 버튼을 제공합니다.
    - CSS 변수를 사용하여 색상 테마를 관리합니다.
- **레이아웃:**
    - 모든 주요 콘텐츠(제목, 게임 영역, 결과, 기록)는 화면 중앙에 수직으로 정렬됩니다.
    - 모바일, 태블릿, 데스크톱 등 모든 화면 크기에서 잘 보이도록 반응형으로 디자인됩니다.
- **폰트 및 스타일:**
    - 가독성이 좋은 모던한 폰트를 사용합니다.
    - "ZeroTap", "No Time to Think", "How Fast Are You?" 문구를 적절한 크기와 스타일로 배치하여 게임의 정체성을 강조합니다.

## 3. 기술 스택

- **프레임워크 없음(Framework-less):** 순수 HTML, CSS, JavaScript (ES6+)를 사용합니다.
- **HTML5:** 시맨틱 마크업으로 구조를 정의합니다.
- **CSS3:**
    - **Flexbox:** 중앙 정렬 및 기본 레이아웃을 위해 사용합니다.
    - **CSS 변수(Custom Properties):** 테마 관리를 위해 사용합니다.
    - **미디어 쿼리:** 반응형 디자인을 위해 사용합니다.
- **JavaScript (ES6+):**
    - **DOM 조작:** 게임의 인터랙션을 처리합니다.
    - **`setTimeout`:** 무작위 지연 시간을 구현합니다.
    - **`Date.now()`:** 정밀한 시간 측정을 위해 사용합니다.
    - **`localStorage`:** 최고 기록을 영구적으로 저장하기 위해 사용합니다.

## 4. 개발 단계별 계획

1.  **HTML 구조 설계 (`index.html`):**
    - 게임 제목, 문구, 게임 화면, 결과 표시, 최고 기록, 테마 전환 버튼 등 모든 UI 요소에 대한 기본 마크업을 작성합니다.

2.  **CSS 스타일링 (`style.css`):**
    - 기본 레이아웃과 중앙 정렬 스타일을 적용합니다.
    - 다크/라이트 모드에 대한 색상 팔레트를 CSS 변수로 정의합니다.
    - 버튼, 텍스트, 게임 영역 등 각 요소의 스타일을 디자인합니다.
    - 미디어 쿼리를 사용하여 모바일 화면에 맞게 스타일을 조정합니다.

3.  **JavaScript 로직 구현 (`main.js`):**
    - **1단계 (기본 로직):**
        - DOM 요소 참조를 변수에 할당합니다.
        - 게임 상태(대기, 준비, 측정 등)를 관리할 변수를 초기화합니다.
        - 화면 클릭 이벤트를 감지하여 게임을 시작하고 결과를 측정하는 핵심 로직을 구현합니다.
    - **2단계 (기록 시스템):**
        - `localStorage`에서 최고 기록을 불러오고, 게임 결과와 비교하는 로직을 추가합니다.
        - 최고 기록 경신 시 이름을 입력받는 UI를 활성화하고, 새로운 기록을 저장 및 표시하는 기능을 구현합니다.
    - **3단계 (부가 기능):**
        - 테마 전환 버튼에 이벤트 리스너를 추가하여 `body` 요소의 클래스를 토글하고 CSS 변수를 통해 테마를 변경하는 기능을 구현합니다.
